업무 관리 시스템 사용자 매뉴얼

개인 업무 관제판의 모든 기능을 안내합니다.
버전: 1.0 대상: 일반 사용자 업데이트: 2026-05-15

1개요

본 시스템은 개인 업무를 체계적으로 관리하기 위한 웹 기반 업무 관제판입니다. 단순 To-Do 앱이 아니라, 장애 로그 분석·충전기 연동 검증·제조사 문의·펌웨어 업데이트·인증 시험·고객 민원 같은 실무 업무 흐름을 추적하기 위해 설계되었습니다.

주요 기능

  • 대시보드 — 오늘 처리해야 하는 업무와 주요 지표를 한눈에 확인
  • 업무 목록 — 검색·필터·정렬로 전체 업무를 관리
  • 칸반 보드 — 상태별 흐름을 카드 드래그로 직관적으로 조정
  • 캘린더 — 월/주/일 단위로 일정 확인, 범위 막대, 드래그&드롭
  • 프로젝트 — 업무를 프로젝트 단위로 묶어 관리
  • 회신 대기함 — 외부 회신을 기다리는 업무의 대기 일수를 자동 집계
  • 데이터 백업/복원 — JSON 파일로 안전하게 보관
💡 팁 본 매뉴얼은 좌측 목차로 빠르게 이동할 수 있습니다. 자주 사용하는 기능은 키보드 단축키 섹션을 참고하세요.

2시작하기

2.1 로그인

2.2 처음 사용한다면

업무 데이터가 비어 있다면 대시보드에 안내 카드가 나타납니다. 두 가지 방법으로 시작할 수 있습니다.

  • 업무 추가 — 빈 화면에서 「업무 추가」 버튼을 눌러 수동으로 입력
  • 샘플 데이터 불러오기 — 「설정」 메뉴에서 미리 준비된 샘플 데이터로 화면 구성을 미리 확인
📌 안내 샘플 데이터는 학습용입니다. 실제 사용 전 「설정 → 전체 삭제」로 초기화한 뒤 본인 업무를 입력하세요.

  • 오늘 해야 할 업무 → 클릭 시 「업무 목록」으로 이동

  • 최근 업데이트 업무 → 최근 수정된 진행 중 업무 5건

5.1 빠른 필터

화면 상단의 알약 모양 버튼으로 자주 쓰는 분류에 즉시 전환합니다.

  • 전체 / 오늘 할 일 / 이번 주 마감 / 지연 업무 / 회신 대기 / 긴급 업무
  • 완료 숨기기 토글 — 완료·취소된 업무를 결과에서 제외
  • 각 버튼 우측의 작은 배지는 해당 분류에 해당하는 업무 건수

5.2 검색 + 드롭다운 필터

키워드 검색은 제목·설명·다음 액션·담당자를 동시에 검색합니다. 드롭다운으로 상태/우선순위/카테고리/프로젝트를 추가로 좁힐 수 있습니다.

5.3 열 정렬 (▲▼)

테이블 각 열 헤더의 ▲▼ 아이콘 버튼을 누르면 해당 열로 정렬됩니다. 한 번 더 누르면 오름차순↔내림차순이 토글됩니다.

💡 팁 제목 텍스트가 아니라 아이콘 버튼을 눌러야 정렬됩니다. 기본 정렬은 마감일 오름차순(▲)이며, 마감일이 없는 업무는 맨 뒤로 정렬됩니다.

5.4 컬럼 필터 (≡)

각 열 헤더의 ≡ 깔때기 아이콘을 누르면 해당 열의 실제 데이터 기준 고유값 목록이 드롭다운으로 나타나며, 클릭 한 번으로 빠르게 필터링됩니다. 활성화된 컬럼 필터는 아이콘이 파란색으로 표시됩니다.

  • 지원 컬럼: 상태, 우선순위, 카테고리, 프로젝트, 담당자
  • 프로젝트 필터에는 "미지정" 옵션도 포함됩니다 (프로젝트가 없는 업무).
  • 해제하려면 드롭다운에서 "전체"를 선택하세요.

5.5 빠른 업무 추가

대시보드와 동일하게, 검색칸 아래 제목만 입력하고 Enter로 즉시 새 업무 생성이 가능합니다.

5.6 행 액션

각 행을 클릭하면 상세 보기가 열립니다. 우측 끝의 ⋯ 메뉴에서는 상태 변경(진행 중/회신 대기/모니터링/지연), 완료 처리, 수정, 삭제를 바로 실행할 수 있습니다.

6.1 칼럼 구성

좌→우 순서: 신규 → 진행 중 → 회신 대기 → 모니터링 → 지연 → 검토 필요 → 보류 → 완료 → 취소. 「완료」와 「취소」 칼럼은 의도적으로 흐리게(투명도 50%) 표시되어 활성 업무에 시선이 가도록 합니다.

6.2 드래그&드롭으로 상태 변경

  • 카드를 누른 채로 다른 칼럼 위로 끌면, 해당 칼럼이 강조 표시되고 카드 자체는 반투명으로 변합니다.
  • 놓으면 즉시 상태가 변경되며, 활동 로그에 자동으로 기록됩니다.
  • 같은 칼럼 내부에서 놓으면 변경 없이 원위치합니다.

6.3 카드 액션

카드를 클릭하면 상세 보기가 열립니다. 카드 우측의 ⋯ 메뉴에서 수정·삭제·복제 등 기존 메뉴를 그대로 사용할 수 있습니다.

7.2 뷰 모드 3종

  • 월 뷰(기본) — 6주 × 7일 그리드. 한 달을 한눈에.
  • 주 뷰 — 1주일 × 7일, 셀이 세로로 크게 확장되어 더 많은 업무가 보임.
  • 일 뷰 — 단일 날짜의 모든 업무를 카드 형태로 자세히 표시. "이 날짜에 추가" 버튼 포함.

7.3 빈 날짜에 빠르게 추가하기

7.4 범위 막대 (시작일 ~ 마감일)

상단 툴바의 「범위 막대」 체크박스를 켜면, 시작일과 마감일이 모두 있는 업무가 가로 막대로 여러 날짜에 걸쳐 표시됩니다.

  • 시작 셀은 왼쪽 둥근 모서리, 끝 셀은 오른쪽 둥근 모서리로 시각적 시작/끝을 표시합니다.
  • 겹치지 않는 업무는 같은 lane을 공유하므로 시각적 정렬이 일관됩니다.
  • 시작일·마감일 중 하나만 있는 업무는 기존처럼 단일 칩으로 표시됩니다.

7.5 드래그&드롭으로 마감일 변경

단일 날짜 칩을 다른 날짜로 드래그하면 현재 활성 "기준" 필드가 그 날짜로 자동 업데이트됩니다.

  • 기준이 마감일이면 마감일이 이동합니다.
  • 기준이 시작일이면 시작일만, 팔로업일이면 팔로업일만 이동합니다.
  • 범위 막대는 드래그 불가입니다. 시작일 또는 마감일을 변경하려면 "범위 막대"를 끄거나 업무를 클릭해 직접 수정하세요.

7.6 셀당 표시 한도

월 뷰에서는 한 셀에 최대 3개까지의 업무 칩이 보이고, 그 이상은 「+N개 더 보기」로 축약됩니다. 클릭하면 해당 날짜의 모든 업무가 팝업으로 표시됩니다.

7.7 연도 선택 팝업

툴바의 「YYYY년」 버튼을 클릭하면 현재 연도 ±12년 범위(총 25년)의 그리드가 나타납니다. 원하는 연도를 클릭하면 같은 달·일을 유지한 채 그 연도로 점프합니다.

💡 팁 키보드 ← →로 빠른 이동, T로 오늘 복귀, M/W/D로 뷰 전환이 가능합니다. 단, 입력칸이나 다이얼로그가 열려 있을 때는 단축키가 동작하지 않습니다.

10설정

데이터 백업, 복원, 샘플 데이터 불러오기, 카테고리 관리 등 시스템 환경을 다루는 페이지입니다.

10.1 JSON 백업 (내보내기)

현재 사용자의 모든 데이터(업무·프로젝트·활동 로그·카테고리)를 하나의 JSON 파일로 다운로드합니다. 정기적으로 백업해두면 안전합니다.

💡 권장 매주 금요일에 백업 파일을 다운로드해 두는 습관을 들이시면 좋습니다. 파일명에 날짜가 포함되어 있어 시간순으로 관리할 수 있습니다.

10.2 JSON 복원 (가져오기)

이전에 내보낸 JSON 파일을 선택하면 그 시점의 데이터로 복구됩니다.

⚠️ 주의 복원은 현재 데이터를 덮어쓰는 작업입니다. 복원 전, 현재 데이터를 먼저 백업해두세요. 실수로 다른 사람의 백업 파일을 불러오면 본인 데이터가 사라질 수 있습니다.

10.3 샘플 데이터 불러오기

처음 사용하거나 데모가 필요할 때 미리 준비된 샘플 업무·프로젝트를 한 번에 불러옵니다. 기존 데이터에 덧붙여 추가됩니다(삭제 아님).

10.4 전체 삭제

모든 업무·프로젝트·로그를 삭제합니다. 확인 다이얼로그가 두 번 표시됩니다.

⚠️ 위험 삭제된 데이터는 복구할 수 없습니다. 반드시 사전에 JSON 백업을 받아두세요.

10.5 카테고리 관리

업무 분류에 사용되는 카테고리를 추가/수정/삭제합니다. 기본 카테고리는 다음과 같습니다.

  • 장애 로그 분석, 충전기 연동 검증, 펌웨어, 제조사 문의, 고객/민원 대응
  • 인증/시험, 프로젝트 관리, 내부 보고, 문서 작성, 기타

14자주 묻는 질문

Q1. 업무 설명에 줄바꿈을 했는데 한 줄로 표시됩니다.

저장 시 줄바꿈은 그대로 보존됩니다. 만약 한 줄로 보인다면 브라우저 캐시 문제일 수 있으니 새로고침을 시도해주세요.

  • Mac — ⌘ Cmd + R
  • Windows — Ctrl + R 또는 F5

Q2. 칸반에서 카드를 옮겼는데 다음 접속 시 원위치되어 있습니다.

드래그&드롭은 서버에 즉시 반영됩니다. 인터넷 연결이 끊긴 상태였다면 변경이 저장되지 않을 수 있습니다. 우상단의 연결 상태를 확인해주세요.

Q3. 캘린더에서 같은 업무를 끌어다 놓아도 변하지 않습니다.

같은 날짜로 드롭하면 변경이 없습니다. 또한 범위 막대로 표시된 업무는 의도적으로 드래그가 불가능합니다. 마감일을 옮기려면 「범위 막대」 체크박스를 끄고 다시 시도하세요.

Q4. 업무를 삭제했는데 활동 로그에는 남아 있습니다.

의도된 동작입니다. 누가 언제 어떤 업무를 삭제했는지 감사(audit) 목적으로 로그는 보존되며, 삭제된 업무 정보는 "(삭제된 업무)"로 표시됩니다.

Q5. 컴퓨터를 바꾸면 데이터는 어떻게 옮기나요?

본 시스템은 클라우드 기반(Supabase)이므로 같은 계정으로 어디서나 로그인하면 동일한 데이터를 볼 수 있습니다. 별도의 이관 작업은 필요하지 않습니다.

Q6. 동일한 업무를 매주 반복 등록해야 하는데 효율적인 방법은?

현재 자동 반복 기능은 없습니다. 대신 상세 보기의 「복제」 버튼으로 빠르게 새 업무를 만들 수 있습니다. 정기 업무가 많다면 백업 JSON을 템플릿처럼 활용하는 방법도 있습니다.

Q7. 모바일에서도 사용 가능한가요?

레이아웃은 데스크톱 우선이지만, 모바일 브라우저에서도 접속·기본 조작이 가능합니다. 칸반 드래그&드롭과 캘린더 단축키는 데스크톱 사용을 권장합니다.

Q8. 데이터를 잃어버렸어요. 복구할 수 있나요?

최근에 백업한 JSON 파일이 있다면 「설정 → 복원」으로 되돌릴 수 있습니다. 백업이 없는 경우 복구가 어려우니, 정기 백업을 강력히 권장합니다.
